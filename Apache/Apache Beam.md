# Apache Beam

Unified programming model for batch and streaming data processing.

https://beam.apache.org/

## Position

- Portable processing API rather than one fixed execution engine.
- Pipelines can run on runners such as Google Dataflow, Apache Flink, Apache Spark, and others.
- Useful when the processing model should be decoupled from the runtime.

## Notes

- Supports event-time processing, windows, triggers, and stateful processing.
- Common in cloud-native stream and batch pipelines.
- Compare with direct use of [Spark](./Apache%20Spark.md), [Flink](./Apache%20Flink.md), and [Google Dataflow](../Google/Google%20Dataflow.md).
